# Breakdown
Example input:
```q
x:"\n"vs"initial state: #..#.#..##......###...###\n\n...## => #\n..#.. => #\n.#... => #"
x,:"\n"vs".#.#. => #\n.#.## => #\n.##.. => #\n.#### => #\n#.#.# => #\n#.### => #\n##.#. => #"
x,:"\n"vs"##.## => #\n###.. => #\n###.# => #\n####. => #"
```

## Common

### d12init
This function parses the input and returns the rules and initial state.

We extract the textual representation of the initial state by cutting the first line on spaces and
keeping only the last element:
```q
q)init:last" "vs first x
q)init
"#..#.#..##......###...###"
```
We create a dictionary of the rules by splitting all rows except the first two on `" => "` and
putting the first part in the key and the last part in the value:
```q
q)rules:{x[;0]!x[;1;0]}" => "vs/:2_x
q)rules
"...##"| #
"..#.."| #
".#..."| #
".#.#."| #
".#.##"| #
".##.."| #
".####"| #
"#.#.#"| #
"#.###"| #
"##.#."| #
"##.##"| #
"###.."| #
"###.#"| #
"####."| #
```
We create an initial state tuple for the next helper function (see below for details):
```q
q)state:(0;init;0;0;1b)
```

### d12step
This function performs a single step of the cellular automaton. It takes the rules and a state tuple
similar to the one generated by `d12init`. The tuple consists of:
- a position offset to indicate where the plant array begins (this avoids adding empty spaces at the
beginning of the array)
- the plant array
- the generation number
- the delta of the position (the first element)
- whether the plants changed

The function simply calculates these values sequentially.

We expand the plant array by two pots each side, plus two pots for "context" so we can apply the
rules:
```q
q)pos:-2+state 0
q)pos
-2
q)plants:"....",state[1],"...."
q)plants
"....#..#.#..##......###...###...."
```
We increment the generation number:
```q
q)gen:1+state 2
q)gen
1
```
We generate subsequences of length 5 via iteration:
```q
q)sect:4_{5#1_x,y}\[" ";plants]
q)sect
"....#"
"...#."
"..#.."
".#..#"
"#..#."
..
```
We generate the new states of the plants by indexing into the rule dictionary with the subsequences:
```q
q)newplants:"."^rules sect
q)newplants
"..#...#....#.....#..#..#..#.."
```
We would like to trim the plant array to avoid the empty pots from constantly extending. To thus
end, we find the indices of the first and last filled pots:
```q
q)ends:{(first x;last x)}where "#"=newplants
q)ends
2 26
```
We shift the position by the starting index and cut off the empty portions of the plant array:
```q
q)pos+:ends 0
q)pos
0
q)newplants2:ends[0]_(1+ends[1])#newplants
q)newplants2
"#...#....#.....#..#..#..#"
```
We find the position delta:
```q
q)posDelta:pos-state 0
q)posDelta
0
```
We check if the plants changed from the previous generation:
```q
q)changed:not state[1]~newplants2
q)changed
1b
```
We assemble the variables into the new state, which is the return value of the function:
```q
q)state:(pos;newplants2;gen;posDelta;changed)
q)state
0
"#...#....#.....#..#..#..#"
1
0
1b
```

## Part 1
We generate the initial state and call the step function with the overload of `/` that takes a
number of iterations and an initial state:
```q
q)rs:d12init x
q)endstate:d12step[rs 0]/[20;rs 1]
q)endstate
-2
"#....##....#####...#######....#.#..##"
20
0
1b
```
The answer is the sums of the positions of the `"#"` characters in the plant array, adding the
offset to each:
```q
q)where "#"=endstate 1
0 5 6 11 12 13 14 15 19 20 21 22 23 24 25 30 32 35 36
q)endstate[0]+where "#"=endstate 1
-2 3 4 9 10 11 12 13 17 18 19 20 21 22 23 28 30 33 34
q)sum endstate[0]+where "#"=endstate 1
325
```

## Part 2
This requires finding a hidden pattern in the input that allows cutting out the instane amount of
iteration. The pattern is that at some point the rules stop changing the plant layout and each new
generation is exactly the same as the previous one but shifted in one direction or the other. This
is why some of the fields in the state tuple look the way they do. We use the overload of `/` that
takes a condition and an initial state, and iterates the function until the condition applied to the
accumulator returns false. In this case the condition is `last`, which extracts the "changed" field
from the state tuple, so we stop when the "shifting" phase has begun:
```q
q)rs:d12init x
q)endstate:d12step[rs 0]/[last;rs 1]
q)endstate
13
"#.#.....#.#.....#.#.........#.#.....#.#..........#.#.....#.#....#.#........#.#......#.#"
87
1
0b
```
Since we store the generation number in the tuple, we can calculate how many generations are left:
```q
q)limit:50000000000
q)gensLeft:limit-endstate 2
q)gensLeft
49999999913
```
And since we also store the position delta, we can calculate the final value of the offset after
this many generations:
```q
q)endpos:endstate[0]+gensLeft*endstate 3
q)endpos
49999999926
```
The calculation of the answer is then similar to the one in part 1:
```q
q)sum endpos+where "#"=endstate 1
999999999374
```
(There is no example for part 2, probably in order to not give a hint on the hidden pattern. This
result is obtained by applying the part 2 solution to the part 1 example.)
